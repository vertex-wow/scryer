# Atlas Scale Factors

## Background

WoW ships two tiers of atlas textures: standard ("1×") and high-DPI ("2×", suffix `-2x` in
our manifest). The `-2x` sheets have twice the physical pixel density; their atlas region
dimensions must be divided by the correct factor to obtain logical WoW UI unit values.

Our atlas manifest is generated by `atlas-gen.ts` from two DB2 tables:
`UiTextureAtlas` and `UiTextureAtlasMember`. Many atlas families appear only in their `-2x`
variant (no 1× entry). Without additional information the fallback divides all dimensions by 2.

## The Authoritative Source: `OverrideWidth` / `OverrideHeight` + `UiCanvas`

### `UiTextureAtlasMember` columns

| Column           | Meaning                                                                               |
| ---------------- | ------------------------------------------------------------------------------------- |
| `OverrideWidth`  | Explicit display width for this atlas member, **in canvas-pixel units** (0 = not set) |
| `OverrideHeight` | Explicit display height, same units (0 = not set)                                     |
| `UiCanvasID`     | Which UiCanvas the Override values are expressed in (0 = inherit from atlas)          |

### `UiCanvas` table

| ID  | Width | Height | Meaning                          |
| --- | ----- | ------ | -------------------------------- |
| 1   | 1024  | 768    | Standard resolution (1× density) |
| 2   | 2048  | 1536   | High-DPI (2× density)            |

### `UiTextureAtlas` columns used

| Column       | Meaning                                                                          |
| ------------ | -------------------------------------------------------------------------------- |
| `UiCanvasID` | Canvas the atlas is designed for (used as fallback when member's UiCanvasID = 0) |

### Conversion formula

`OverrideWidth` is in **canvas pixels**, not WoW UI logical units. WoW's UI coordinate
system has UIParent = 768 units tall = 1024 canvas pixels (UiCanvas 1). Conversion:

```
logicalW = OverrideWidth × (1024 / canvasWidth)
         where canvasWidth = member.UiCanvasID != 0
               ? UiCanvas[member.UiCanvasID].Width
               : UiCanvas[atlas.UiCanvasID].Width
```

When `OverrideWidth = 0`: no override, fall back to `physicalWidth ÷ 2`.

The correct pixel→logical divisor is then:

```
scaleDivisor = logicalW > 0 ? physicalWidth / logicalW : 2
```

## DiamondMetal — Worked Example

Confirmed from wago.tools live DB2 data:

```
UiTextureAtlas 1502 — body corners + top/bottom edges:
  AtlasWidth=256, AtlasHeight=1024, UiCanvasID=1

UiTextureAtlasMember ui-frame-diamondmetal-cornertopleft-2x:
  Width=128, Height=128, OverrideWidth=64, OverrideHeight=64
  UiTextureAtlasID=1502, UiCanvasID=2 (member explicitly overrides to canvas 2)

UiTextureAtlas 1505 — header pieces:
  AtlasWidth=256, AtlasHeight=512, UiCanvasID=2

UiTextureAtlasMember ui-frame-diamondmetal-header-cornerleft-2x:
  Width=128, Height=156, OverrideWidth=64, OverrideHeight=78
  UiTextureAtlasID=1505, UiCanvasID=0 (inherits atlas canvas = canvas 2)
```

Both resolve via canvas 2 (width=2048):

```
logicalW = 64 × (1024 / 2048) = 32
logicalH for header = 78 × (1024 / 2048) = 39
divisor = 128 / 32 = 4
```

Cross-check: `DialogHeaderTemplate` XML hard-codes `<Size x="32" y="39"/>` on these corners.
`128 ÷ 4 = 32` and `156 ÷ 4 = 39` — exact match. ✓

The body border corners are therefore **32×32 logical WoW units** (not 64×64).

## Metal Corners — Not Overridden

```
UiTextureAtlasMember ui-frame-metal-cornertopleft-2x:
  Width=150, OverrideWidth=0, UiCanvasID=2
```

`OverrideWidth=0` → no override → fallback divisor = 2 → logical = 150/2 = **75 WoW units**.
Consistent with `ButtonFrameTemplateNoPortrait` rendering (verified by ExampleFrameTitleFrame tests).

## Implementation

`atlas-gen.ts` now reads `OverrideWidth`, `OverrideHeight`, `UiCanvasID` from the member CSV,
plus `UiCanvasID` from the atlas CSV. It computes the canvas-aware logical size and stores it
as `logicalW` / `logicalH` in the manifest.

Both lookup sites use:

```ts
scaleDivisor = entry.logicalW > 0 ? entry.width / entry.logicalW : 2;
```

Files: `src/assets/atlas-manifest.ts`, `src/lua/api/retail/C_Texture.ts`.
